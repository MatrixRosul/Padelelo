# MASTER PROMPT ДЛЯ AI ДИЗАЙНУ

Ти Senior Product Designer + Art Director + Mobile UX Lead.
Твоє завдання: створити візуально сильний, дуже сучасний, преміальний редизайн мобільного застосунку **PadelElo** (React Native, iOS + Android) з акцентом на **зелений колір** і **білий фон**.

## 1) Головна мета
Створи цілісний сучасний дизайн-сет для всіх екранів, які вже є в продукті, щоб:
- інтерфейс виглядав преміально та впізнавано, а не шаблонно;
- навігація була очевидною і швидкою;
- дані про рейтинг, матчі й турніри читались легко;
- адмін-функції виглядали структуровано і не перевантажували UI;
- стиль був сучасний, легкий, «чистий», з живою пластикою карток і м’якою глибиною.

## 2) Стиль і візуальний напрям
Обов’язкові правила:
- База: світлий інтерфейс, білий/майже білий фон.
- Домінанта: зелена палітра (від глибокого спортивного зеленого до свіжого лайм-акценту).
- Дозволені відхилення: нейтральні сірі, невеликі акценти теплих кольорів для CTA/статусів.
- Заборонено: фіолетова домінанта, темна тема як основна, «плаский нудний» вигляд.

Візуальна мова:
- Поєднай спортивну енергію + преміальну чистоту.
- Використай 1 сміливий display-шрифт для hero-заголовків + читабельний UI-шрифт для всіх даних.
- Додай легку об’ємність: тіні, глибину, blur-ефекти, мікроградієнти, але без перевантаження.
- Компоненти мають мати відчутну ієрархію: hero, секції, картки, бейджі, кнопки, стани.

## 3) Бренд-атмосфера
Ключові слова: court energy, precision, momentum, confidence, premium sports tech.

Бажаний настрій:
- швидкість;
- контроль;
- прогрес;
- здоровий азарт;
- командність.

## 4) Дизайн-токени (запропонуй конкретні значення)
Побудуй систему токенів:
- Color tokens:
  - primary green (main CTA, active states)
  - primary dark green (headers, strong emphasis)
  - green container/surface tints
  - success, warning, error
  - neutral text scale (high/medium/low contrast)
  - border/divider colors
  - elevated surfaces
- Typography tokens:
  - display xl / lg
  - title / section
  - body / caption / overline
  - button label styles
- Spacing scale: 4/8/12/16/20/24/32...
- Radius scale: card/button/chip/pill
- Shadow/elevation tokens
- Motion tokens (duration/easing)

## 5) Обов’язкові UX принципи
- Mobile-first: ідеально для 375px ширини, адаптація до більших девайсів.
- Safe areas, великі tap-зони, доступність і читабельність.
- Контраст, стан disabled/loading/error/success для всіх action-компонентів.
- Довгі імена гравців/клубів не ламають макет.
- Таблиці/рейтинги не мають виглядати «важкими».
- Складні адмін-флоу мають бути візуально спрощені через секції та пріоритетні дії.

## 6) Повний список екранів (зроби детальний дизайн для кожного)

### 6.1 Auth Loading / Session Check
Стан завантаження сесії:
- чистий екран з бренд-маркером;
- індикатор завантаження;
- коротке повідомлення.

### 6.2 Login / Register Screen
Поточна структура:
- hero-блок з назвою продукту і підзаголовком;
- переключення режимів: «Увійти» / «Створити акаунт»;
- форми для login + password;
- форми реєстрації: full name, display name, login, password, confirm password;
- помилки валідації;
- helper-блок з тестовими акаунтами;
- primary CTA кнопка.

Що потрібно в дизайні:
- виразний entry experience;
- ідеальна візуальна ієрархія форми;
- premium-inputs, стан focus/error/success;
- сучасний switch між login/register;
- стильна, але не криклива hero-секція.

### 6.3 Dashboard Screen
Поточні блоки:
- top bar з брендом і нотифікаціями;
- search-like action «Find players...»;
- велика invite-card / tactical card;
- стани профілю (need linked player / loading / error);
- секція recent activity (останні матчі, outcome W/L/D, timestamp);
- секція recommended rivals (горизонтальний скрол).

Що потрібно в дизайні:
- сильний dashboard-hero;
- пріоритизація ключових дій;
- стильний feed останніх активностей;
- картки суперників з акцентом на Elo та персоналії.

### 6.4 Leaderboard Screen
Поточні блоки:
- hero «Ranking»;
- podium top-3;
- full rankings list;
- пошук гравця;
- load more;
- поточний користувач виділений у списку.

Що потрібно в дизайні:
- wow-подіум без перевантаження;
- рейтинг, що читається миттєво;
- чіткі акценти на #позицію, Elo, W/D/L;
- продуманий стан search/filter.

### 6.5 Matches Screen
Поточні блоки:
- hero «Match Center»;
- summary cards (win rate, completed, record);
- список минулих матчів;
- outcome статуси;
- деталізація scoreline;
- load more.

Що потрібно в дизайні:
- фокус на динаміці сезону;
- зрозумілі результати матчу в 1 погляд;
- сильна типографічна структура для статистики.

### 6.6 Tournaments Screen
Поточні блоки:
- hero + список доступних турнірів;
- для admin: форма створення турніру (назва, клуб, тип, дати, час, дедлайн, max players);
- date/time picker interactions;
- cards турнірів із категоріями і статусами open/closed.

Що потрібно в дизайні:
- зробити сторінку зрозумілою для двох ролей (гравець/адмін);
- admin-блоки відділити візуально і логічно;
- карточки турнірів зробити «живими» й інформативними;
- дедлайн реєстрації виділити максимально читабельно.

### 6.7 Tournament Details Screen (найскладніший екран)
Поточні блоки/таби:
- Hero турніру (назва, тип, статус, стан реєстрації);
- Tabs: participants, matches, results, rating, groups (для GROUP_STAGE), bracket (для PLAYOFF);
- Реєстрація гравця + countdown дедлайну;
- Admin lifecycle actions: publish/open/close/start/restart/cancel;
- Admin manage registrations (search + add player + confirm/remove);
- Match management: ручний ввід сетів, winner side, simulate;
- Results standings;
- Rating changes list;
- Categories list.

Що потрібно в дизайні:
- складний функціонал зробити легким для сприйняття;
- розвести user actions і admin actions по пріоритету;
- максимально чіткі таби та інформаційна архітектура;
- сильні стани для success/error/confirmation;
- структура, де помилитися в критичних діях складно.

### 6.8 Profile Screen
Поточні блоки:
- account strip + sign out;
- player profile summary (графік Elo, інсайти, recent matches);
- для admin: panel з quick actions;
- створення клубу;
- список існуючих клубів;
- редагування гравця.

Що потрібно в дизайні:
- зробити профіль персональним і статусним;
- data-візуалізацію читабельною та сучасною;
- admin panel оформити як керовану «control zone».

### 6.9 Player Details Screen
Поточні блоки:
- повторно використовує профільну аналітику гравця;
- loading / error / retry.

Що потрібно в дизайні:
- уніфікувати з Profile, але зробити відчуття «перегляду іншого гравця»;
- акцент на продуктивність гравця, progression і форму.

### 6.10 Глобальні навігаційні елементи
- Floating bottom tabs: Dashboard, Leaderboard, Matches, Tournaments, Profile.
- Top bar: бренд, іконка нотифікацій, опційний badge.

Що потрібно в дизайні:
- преміальна tab bar з сучасною формою;
- активний таб має виглядати виразно;
- top bar має бути легким, не «важким».

## 7) Стани для кожної сторінки
Обов’язково опиши дизайн для:
- default/content state;
- loading state;
- empty state;
- error state;
- success feedback (де релевантно);
- disabled state для кнопок;
- pressed/hover/focus (mobile-еквіваленти).

## 8) Motion і мікроанімації
Додай продуману анімацію:
- м’який page-enter;
- stagger reveal для карток;
- таб-перемикання з коротким smooth transition;
- feedback анімації для success/error;
- countdown/live елементи без «стресового» мерехтіння.

## 9) Accessibility
- Контраст WCAG-friendly для основного тексту і кнопок.
- Не покладатися тільки на колір для статусів W/L/D.
- Розмір шрифтів і кнопок дружній до мобільного використання.
- Важливі admin-дії мають підтвердження і чіткі risk cues.

## 10) Технічні обмеження
- Це дизайн для React Native застосунку.
- Не пропонувати web-only патерни, які погано працюють у native mobile.
- Зберігати реалістичну складність компонентів.
- Не змінювати бізнес-логіку, лише UX/UI подачу.

## 11) Що саме повернути у відповіді
Поверни результат у такому форматі:

1. Design Concept (2-3 варіанти в одному стилістичному напрямі).
2. Color System (конкретні hex-токени, де primary — зелений, фон — білий).
3. Typography Pairing (конкретні шрифти + ролі).
4. Component Library (buttons, inputs, cards, tabs, pills, badges, lists, charts).
5. Детальний екранний опис КОЖНОЇ сторінки з пункту 6.
6. State Matrix (loading/empty/error/success/disabled по сторінках).
7. Motion Spec (duration/easing/що анімується).
8. Мобільна адаптація (малий/середній/великий екран).
9. Пріоритетний план впровадження (MVP → polish).

## 12) Додатковий креативний бриф
Зроби дизайн так, щоб він виглядав як сучасний sports-tech продукт рівня топових мобільних застосунків 2026 року.
Хочу сміливий, чистий, енергійний інтерфейс, у якому зелений + білий працюють як сильна бренд-пара.
Відхилення дозволені, якщо вони підсилюють якість і не руйнують базову концепцію.

Фінальний результат має бути не «типовим шаблоном», а впізнаваною дизайн-системою з характером.
